# 生产部署手册与操作流程

适用于：**阿里云 ECS**、与现有 **8000 端口 Python 服务** 共存、**共用同一台 MySQL**、**Nginx**（可配合 **宝塔**）、域名规划为 **`*.ok.sourcefire.cn`**。

本仓库为 **FastAPI 后端**；会员端 H5、管理端等前端在 **其它仓库**，需单独构建上传。

相关文件：

- Nginx 配置示例：[deploy/nginx-ok.sourcefire.cn.example.conf](../deploy/nginx-ok.sourcefire.cn.example.conf)
- 接口说明：[前端接口联调说明.md](前端接口联调说明.md)

---

## 〇、部署前准备（核对清单）

| 项 | 说明 |
|----|------|
| ECS | 已可 SSH 登录；安全组将放行 **80、443**（**不必**对公网开放 **8001**） |
| 域名 | 已规划子域：`api` / `h5` / `admin`（均在 `ok.sourcefire.cn` 下） |
| MySQL | 本机或 RDS可连；本后端建议使用 **独立库名**（如 `meal_delivery`），与旧项目库分离 |
| 代码 | 本仓库已同步到服务器（`git clone` /上传压缩包均可） |
| 前端 | H5、管理端仓库可在本机或 CI 构建，准备好 `dist` |

**推荐端口**：本 API 使用 **8001**；现有服务继续 **8000**，互不占用。

---

## 一、完整操作流程（按顺序执行）

### 步骤 1：DNS 解析

在域名 DNS 控制台为以下主机记录添加 **A 记录**，指向 **ECS 公网 IP**：

- `api.ok.sourcefire.cn`
- `h5.ok.sourcefire.cn`
- `admin.ok.sourcefire.cn`

等待解析生效（通常数分钟至数小时）。可用 `ping api.ok.sourcefire.cn` 粗测是否指向正确 IP。

---

### 步骤 2：阿里云安全组

在 ECS 安全组规则中：

- **入方向**：允许 **TCP 80**、**TCP 443**（来源按业务设为 `0.0.0.0/0` 或更严）。
- **不要**把 **8001** 暴露给公网（API 仅本机 Nginx 访问即可）。

---

### 步骤 3：MySQL 建库与表结构

在服务器上使用 MySQL 客户端（或与现有库同一实例）执行：

1. 创建数据库（名称与 `.env` 中 `MYSQL_DATABASE` 一致，示例为 `meal_delivery`）：

   ```sql
   CREATE DATABASE IF NOT EXISTS meal_delivery
     DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```

2. 导入表结构（在服务器项目目录下，路径按实际调整）：

   ```bash
   mysql -h 127.0.0.1 -u你的用户 -p meal_delivery < /path/to/admin-okfine/sql/schema.sql
   ```

若库已存在且仅需补表，可按仓库 `sql/patches/` 说明增量执行。

---

### 步骤 4：后端 Python 环境与依赖

在服务器上进入项目目录（示例 `/www/wwwroot/admin-okfine`，按实际修改）：

```bash
cd /path/to/admin-okfine
python3 -m venv .venv
source .venv/bin/activate   # Windows 服务器: .venv\Scripts\activate
pip install -U pip
pip install -r requirements.txt
```

---

### 步骤 5：配置 `.env`

```bash
cp .env.example .env
```

编辑 `.env`，至少确认：

- **`MYSQL_HOST`**：MySQL 在本机则多为 `127.0.0.1`（若 API 跑在 Docker 内访问宿主机 MySQL，**不能**填容器内的 `127.0.0.1`，需填宿主机地址或 `host` 网络，见下文「Docker 注意」）。
- **`MYSQL_USER` / `MYSQL_PASSWORD` / `MYSQL_DATABASE`**：与步骤 3 一致。
- **`JWT_SECRET`**：生产使用足够长的随机串。
- **`CORS_ORIGINS`**：上线后的前端地址，**逗号分隔**、**含协议**，例如：

  ```env
  CORS_ORIGINS=https://h5.ok.sourcefire.cn,https://admin.ok.sourcefire.cn
  ```

- **微信小程序 / 高德**：按 [README.md](../README.md) 与 `.env.example` 填写 `WX_MINI_APPID`、`WX_MINI_SECRET`、`AMAP_KEY` 等；未配置时小程序登录或地理编码会失败或降级。

---

### 步骤 6：初始化管理员与配送员（首次）

在项目目录、已激活 venv：

```bash
python scripts/create_admin.py admin 你的强密码
python scripts/create_courier.py C001 123456 张三
```

账号与参数按实际需要修改。

---

### 步骤 7：本机启动 API（先验证再挂 Nginx）

仅监听本机，避免未配防火墙时直接暴露：

```bash
uvicorn app.main:app --host 127.0.0.1 --port 8001
```

另开 SSH 窗口测试：

```bash
curl -s http://127.0.0.1:8001/health
```

期望返回：`{"status":"ok"}`。确认无误后，用 **Ctrl+C** 停掉临时进程，进入步骤 8 用守护进程长期运行。

---

### 步骤 8：用 systemd 长期运行 API（推荐，不依赖宝塔）

创建 `/etc/systemd/system/okfine-api.service`（路径与 `User` 按实际修改）：

```ini
[Unit]
Description=Okfine FastAPI (uvicorn)
After=network.target mysql.service

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/path/to/admin-okfine
Environment="PATH=/path/to/admin-okfine/.venv/bin"
ExecStart=/path/to/admin-okfine/.venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8001
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

启用并启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable okfine-api
sudo systemctl restart okfine-api
sudo systemctl status okfine-api
```

日志：`journalctl -u okfine-api -f`

**若使用宝塔「Python 项目」**：在面板中指定项目路径、虚拟环境、启动命令与端口 **8001**，效果等同，可替代 systemd。

---

### 步骤 9：Nginx 反代与静态站点

**方式 A — 纯 Nginx**

1. 将 [deploy/nginx-ok.sourcefire.cn.example.conf](../deploy/nginx-ok.sourcefire.cn.example.conf) 为参考，把 `root` 改为服务器上 H5/管理端实际上传目录（如 `/var/www/okfine-h5`）。
2. 放入 `sites-enabled` 或 include 到主配置，`nginx -t` 通过后 `systemctl reload nginx`。

**方式 B — 宝塔**

1. **网站 → 添加站点**：分别添加 `api.ok.sourcefire.cn`、`h5.ok.sourcefire.cn`、`admin.ok.sourcefire.cn`。
2. **API 站点**：在站点配置中加入反代到 `http://127.0.0.1:8001`（或把示例里 `location /` 整段粘贴进「配置修改」）。
3. **H5 / 管理端站点**：网站根目录指向上传的 `dist` 内容；伪静态选择 Vue/React 规则，或手动保证存在：

   ```nginx
   location / {
       try_files $uri $uri/ /index.html;
   }
   ```

---

### 步骤 10：前端构建与上传

在 **H5 仓库**、**管理端仓库** 分别：

1. 将 API 地址配置为 `https://api.ok.sourcefire.cn`（环境变量或配置文件，按各项目约定）。
2. 执行生产构建，例如：`npm run build`。
3. 将输出目录（多为 `dist`）上传到服务器，例如：
   - H5 → `/var/www/okfine-h5`（与 Nginx `root` 一致）
   - 管理端 → `/var/www/okfine-admin`

上传后浏览器访问对应子域，检查页面与接口请求域名是否正确。

---

### 步骤 11：HTTPS 证书

- **宝塔**：各站点 → SSL → Let's Encrypt → 申请并开启「强制 HTTPS」。
- **纯 Nginx**：为各 `server` 配置 `ssl_certificate` / `ssl_certificate_key`，或使用 certbot。

申请前确保 **80** 可从公网访问且域名已解析到本机。

---

### 步骤 12：上线验收清单

| 检查项 | 操作 |
|--------|------|
| 健康检查 | 浏览器或 `curl` 访问 `https://api.ok.sourcefire.cn/health` |
| CORS | 浏览器打开 H5/管理端，登录或调用接口无跨域报错；若有问题，核对 `.env` 中 `CORS_ORIGINS` 是否与页面 Origin 完全一致（https、主机名、无尾斜杠） |
| 数据库 | 登录、菜单、关键业务走通；MySQL 与旧项目库分离则无表冲突 |
| 微信登录/高德 | 按环境实际拨测 `wx/mini/login`、地址地理编码相关接口 |
| 定时任务 | 单机单进程仅一套 APScheduler；若多副本需按 README 调整 |

## 二、架构与端口（摘要）

| 子域名 | 用途 | 说明 |
|--------|------|------|
| `api.ok.sourcefire.cn` | 本后端 | 本机 **127.0.0.1:8001**，Nginx 反代 |
| `h5.ok.sourcefire.cn` | 会员端 | 静态 `dist` + `try_files` |
| `admin.ok.sourcefire.cn` | 管理端 | 同上 |

与 **8000** 服务：**不同端口、不同进程、建议不同 MySQL 库**；共用同一 MySQL **实例**即可。

---

## 三、Docker 访问本机 MySQL 的注意点

若 API 在容器内运行，`MYSQL_HOST=127.0.0.1` 指向的是 **容器自己**。请改为：

- Linux 常见：`172.17.0.1`（docker0 网关），或
- Docker Desktop / 部分环境：`host.docker.internal`，或
- 使用 **`network_mode: host`**（与宿主机共享网络，此时可用 `127.0.0.1`）。

按你实际 Docker 编排选择其一，并保证 MySQL 用户允许从对应来源连接。

---

## 四、APScheduler 说明

多进程或多容器 **重复** 跑同一套定时任务可能导致重复扫描/通知。生产 **单机单 uvicorn 进程** 一般无此问题；水平扩展时请改为单 Scheduler 实例或独立 Worker（详见 [README.md](../README.md)）。

---

## 五、常见问题

**502 Bad Gateway（访问 API 域名）**  
检查 `okfine-api`（或宝塔项目）是否运行、`127.0.0.1:8001` 是否可 `curl`；检查 Nginx 反代 `proxy_pass` 与错误日志。

**前端能打开但接口报 CORS**  
核对 `CORS_ORIGINS` 是否包含当前页面完整 Origin；修改 `.env` 后需 **重启 API 进程**。

**刷新前端子路由 404**  
Nginx 未配置 `try_files ... /index.html`，按步骤 9 补上。

---

部署完成后请再次确认：**`.env` 中 `CORS_ORIGINS`** 与浏览器实际访问的 **https 前端 Origin** 完全一致。
