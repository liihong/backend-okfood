# 生产部署说明

线上环境已就绪。对外仅使用 **`https://ok.sourcefire.cn`** 一个域名，通过 Nginx 路径区分 H5、管理端与 API。

## 环境一览

| 项 | 值 |
|----|----|
| 域名 | **`ok.sourcefire.cn`**（HTTPS） |
| 后端 API | 本机 `127.0.0.1:8001`，Nginx 反代 `/api`、`/static`、`/health` 等 |
| API 服务 | **`okfine-api`** → `systemctl restart okfine-api` |
| 定时任务 | **`okfine-scheduler`** → `python -m app.jobs.worker` |
| 后端代码 | **`/var/www/okfood/backend`** |
| H5 静态 | **`/var/www/okfood/h5`**（站点根路径 `/`） |
| 管理端静态 | **`/var/www/okfood/backend/okfood-admin/dist`**（路径 `/admin`） |

## 路径与用途

| 路径 | 说明 |
|------|------|
| `/` | 会员端 H5 |
| `/dish-catalog/` | 公开菜品库展示页（免登录，`?store_id=`） |
| `/admin` | Web 管理端 |
| `/api/*` | FastAPI 业务接口 |
| `/health` | 健康检查 |

小程序与管理端请求 API 时使用 **`https://ok.sourcefire.cn`**（接口前缀 `/api/...`）。

## 日常更新

在服务器执行：

```bash
bash /var/www/okfood/backend/deploy/update-production.sh
```

脚本会 `git pull`、更新 Python 依赖并重启 **`okfine-api`**（及已安装的 **`okfine-scheduler`**）。重启 `systemctl` 时会自动 `sudo`。

可选环境变量：

| 变量 | 说明 |
|------|------|
| `BUILD_ADMIN=1` | 顺带在服务器构建管理端 `dist` |
| `SKIP_GIT=1` | 跳过 `git pull` |

仅改管理端时，也可手动：

```bash
cd /var/www/okfood/backend/okfood-admin
npm ci && npm run build
```

更新后验收：`curl -s https://ok.sourcefire.cn/health` 应返回 `{"status":"ok"}`。

## 常见问题

**502（访问 `/api`）**  
检查 `okfine-api` 是否运行、`curl http://127.0.0.1:8001/health` 是否正常；查看 Nginx 错误日志。

**接口 CORS 报错**  
`.env` 中 `CORS_ORIGINS` 须包含页面 Origin（如 `https://ok.sourcefire.cn`），修改后重启 **`okfine-api`**。

**前端子路由刷新 404**  
确认 Nginx 对静态站点配置了 `try_files $uri $uri/ /index.html`。

**查看日志**

```bash
journalctl -u okfine-api -f
journalctl -u okfine-scheduler -f
```
